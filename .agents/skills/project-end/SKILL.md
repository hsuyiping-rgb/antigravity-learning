# 收工流程

當使用者說「收工」或「下班了」時，請執行以下步驟：

1. **安全漏洞掃描**：
   - 掃描工作區中是否包含敏感資訊（例如：是否有將 OpenAI API Key `sk-`、Firebase 憑證等明文密碼寫在程式碼中，如果是，提醒使用者移入 `.env`）。
2. **Git 存檔與推送**：
   - 執行 `git add .`。
   - 根據今日的變更歷史自動生成簡潔實用的 commit message。
   - 向使用者確認是否提交，確認後執行 `git commit` 與 `git push`。
3. **更新 Obsidian 每日筆記**：
   - 獲取今天的日期格式（例如：`2026-07-04`）。
   - 如果 `G:\我的雲端硬碟\secondbrain\每日筆記\<YYYY-MM-DD>.md` 不存在，則自動建立它（可以使用 `Templates/` 目錄下的模板或直接寫入 markdown 結構）。
   - 在該文件中更新：
     - **今日進度**：今日完成了什麼工作與變更。
     - **明日待辦**：明日留待處理的下一步計畫。
